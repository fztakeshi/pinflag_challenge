## Supuestos

- El request del tercer endpoint siempre será un JSON de tipo {name: 'name'}
- Para el tercer endpoint, al buscar en la base de datos se busca la coincidencia exacta de nombre y retorna todas las coincidencias. Por otro lado, la API de rick and morty se dejó tal cual como entregan los datos al hacer una query de un nombre (sin coincidencia exacta).
